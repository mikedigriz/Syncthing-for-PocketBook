# Заметки по скриптам

## syncthing_pro.app

- **Логирование**: в логи попадают только ошибки, иначе файл раздуется
  на встроенном хранилище.
- **Статус**: опрашивает Syncthing через REST API и показывает
  актуальное состояние всех папок. Парсинг рассчитан на REST API 2.x.

## Как работает статус

Скрипт опрашивает Syncthing через REST API и показывает реальное положение
по всем папкам сразу. Если хоть одна папка в ошибке или синхронизируется -
это отразится в заголовке. Папки на паузе показываются отдельной строкой
и не меняют общий статус (пауза - это норма, если её сам поставил пользователь).

Опрос идёт только когда демон уже работает и не влияет на его запуск
или остановку.

**Важное уточнение по таймингам**: `timeout -t 3` в `syncthing_pro.app`
откалиброван под быстрый опрос локального API по unix-сокету. Если на вашем
устройстве API отвечает медленнее или у вас много папок, может потребоваться
увеличить это значение, чтобы скрипт не прерывался по таймауту.

## Блокировка

`syncthing_pro.app` при запуске ставит блокировку через
`mkdir /tmp/syncthing.lock`. Это защита от повторного запуска: если папка
уже есть, `mkdir` падает и скрипт выходит, не плодя второй процесс.

Важно: снимает блокировку `syncthing_kill.app` строкой
`rmdir /tmp/syncthing.lock`. Если её убрать, после остановки лок останется
на месте, и `syncthing_pro.app` больше не запустится, пока вы не удалите
папку вручную или не перезагрузите устройство.

## Отладка на устройстве

Чтобы заглянуть внутрь и понять, почему скрипт падает, пригодятся:
- [pb-terminal](https://github.com/CatInBeard/pb-terminal): терминал прямо на ридере.
- [pbsshd](https://www.mobileread.com/forums/attachment.php?attachmentid=123724&d=1402034568): доступ по SSH.
- [rsh](https://www.mobileread.com/forums/showthread.php?t=116350&highlight=pocketbook+rsh): ещё один способ зайти в шелл.
- [root для устройства](https://www.mobileread.com/forums/showthread.php?t=325185): опасно, можно сломать устройство.

Содержимое бинарников смотрите командой `strings`, например
`strings /ebrmain/bin/netagent`. Так удобно подсмотреть, какие команды
и пути доступны в прошивке (особенно полезно на старых моделях,
где набор команд другой).

## Окончание строк в скриптах

> [!IMPORTANT]\
> Все файлы скриптов (`.app`, `.sh`) ДОЛЖНЫ использовать **LF**
> (Line Feed, `\n`), а не CRLF (Carriage Return + Line Feed, `\r\n`).

На Windows по умолчанию Git конвертирует LF в CRLF и обратно, но скрипты
на PocketBook требуют строго LF. Если использовать CRLF, скрипты
не будут работать.

**Как проверить и исправить:**
- **VS Code**: внизу справа нажмите `CRLF` → выберите `LF`
- **Sublime Text**: Menu → View → Line Endings → Unix (LF)
- **Другие редакторы**: посмотрите в их настройки окончаний строк

Git автоматически преобразует окончания в соответствии с
[.gitattributes](.gitattributes), но лучше убедиться в настройках редактора.
